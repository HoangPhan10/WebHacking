Common Web Application Vulnerabilities and Security Risks

Lab1: XSS (Cross Site Scripting)

Malicious scripts can be injected into web pages, allowing attackers to steal cookies or manipulate content.

Objective: Learn how to detect and prevent XSS attacks.

Lab2: SQL Injection

Unsanitized input lets attackers manipulate database queries, exposing or altering sensitive data.

Objective: Practice secure query handling and input validation.

Access the lab

Lab3: LDAP Injection

Crafted input can change LDAP queries, enabling unauthorized access or directory manipulation.

Objective: Understand LDAP query sanitization and secure directory access.

Access the lab

Lab4: Code Injection

User input is executed as code, potentially giving attackers control over application logic.

Objective: Prevent execution of untrusted input in application logic.

Access the lab

Lab5: OS Command Injection

Input is passed to system commands, allowing attackers to run arbitrary commands on the server.

Objective: Learn to sanitize inputs and avoid unsafe system calls.

Access the lab

Lab6: Expression Language Injection

Objective: Secure template engines and validate dynamic expressions.

Objective: Validate email inputs and prevent header manipulation.

Access the lab

Lab8: Null Byte Injection

Access the lab

Lab9: DoS Attack

Objective: Implement resource limits to prevent denial of service.

Objective: Enforce file type restrictions for safe uploads.

Lab11: Open Redirect

Objective: Prevent unsafe redirects and validate destination URLs.

Access the lab

Lab12: Brute-force Login

Without account lockout, repeated login attempts can guess valid credentials.

Objective: Implement account lockout and rate limiting mechanisms.

Access the lab

Lab13: Session Hijacking

Access the lab

Lab14: Session Fixation

Objective: Regenerate session IDs after login to prevent fixation.
