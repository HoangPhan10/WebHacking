Common Application Troubles and Risk Labs

Lab1: Deadlock (Java)

Two or more threads wait on each other’s locks, causing the system to halt.

Objective: Detect cycles, use lock ordering, and employ timeouts.

Access the lab

Lab2: Livelock (Java)

Threads keep responding to each other without progress, consuming CPU but not completing work.

Objective: Introduce backoff and fairness to ensure forward progress.

Access the lab

Lab3: Thread starvation

Low-priority threads never get CPU time due to resource monopolization by others.

Objective: Use fair locks and thread pools with balanced scheduling.

Access the lab

Objective: Identify non-thread-safe components, add synchronization or use immutable types.
